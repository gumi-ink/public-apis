## 参与贡献

我们欢迎社区成员参与贡献！请遵循以下指南：

### 提交准则

* 确保你想添加的 API 有完整的免费访问权限或至少有一个免费套餐
* 不要在 API 名称中包含顶级域名（例如：❌Gmail.com ✔Gmail）
* 不要在 API 名称后面添加 `API` 字样（例如：❌Gmail API ✔Gmail）
* 确保 API 有适当的文档
* 每个 API 条目应按字母顺序排列
* 描述不应超过 100 个字符
* 如果 API 似乎属于多个类别，请将其放在最符合所提供服务的部分

### API 条目格式

| API | Description | Auth | HTTPS | CORS |
| --- | --- | --- | --- | --- |
| [API 名称](文档链接) | API 描述 | 是否需要认证 | 是否支持 HTTPS | 是否支持 CORS |

**Auth 字段可选值：**
* `OAuth` - API 支持 OAuth 认证
* `apiKey` - API 使用私钥字符串/令牌进行认证
* `X-Mashape-Key` - 需要发送的头部名称
* `No` - API 不需要认证
* `User-Agent` - 需要随请求发送的头部

**CORS 字段可选值：**
* `Yes` - API 支持 CORS
* `No` - API 不支持 CORS
* `Unknown` - 未知是否支持 CORS

### 提交 Pull Request

1. [Fork](https://github.com/public-apis/public-apis/fork) 本仓库并在本地克隆
2. 创建一个分支进行编辑
3. 按照上述格式添加你的 API
4. 提交 PR 到 `master` 分支，标题格式为 `Add API-name API`
5. 确保在提交前压缩所有提交

感谢你的贡献！
